## Local dev

```bash

npm install
npm run dev
```

Open `http://localhost:4321`.

Pages live:
- `/` — Homepage
- `/manifesto` — The founding manifesto
- `/manifestos/you-raised-them-now-build-yourself` — First long-form essay

All other nav links return 404 — stubs for future pillar pages.

## Deploy to Vercel

1. Vercel dashboard → New Project → Import `WEBWORKSA1/afterchildren-com`
2. Vercel reads `vercel.json` automatically — build runs from `/site`
3. No env vars needed for initial deploy
4. Add `afterchildren.com` as custom domain when ready

**Build settings (auto-detected via vercel.json):**
- Build command: `cd site && npm install && npm run build`
- Output directory: `site/dist`

---

## Migrating later (Netlify / Hostinger)

The site builds to static HTML/CSS in `site/dist/`. To move:
- **Netlify:** Same repo. Build command `cd site && npm run build`, publish directory `site/dist`
- **Hostinger:** Run `npm run build` locally, upload `site/dist/` contents via SFTP

## Newsletter setup (Beehiiv)

1. Create free Beehiiv account at beehiiv.com
2. Publication: "The AfterChildren Sunday Letter"
3. Settings → Integrations → Embedded form action URL
4. Vercel → Project → Settings → Environment Variables → add:
   ```
   PUBLIC_BEEHIIV_URL=https://...your form endpoint...
   ```
5. Welcome sequence: 5 emails. Day 0, 2, 5, 10, 14.

## The brand law

Before writing or shipping anything, re-read `brand/01-brand-identity-and-voice-guide.md`. Voice consistency is the entire moat. The day you publish a soft listicle is the day the brand starts dying.

The "Would AARP publish this?" test is non-negotiable. If yes, rewrite.
